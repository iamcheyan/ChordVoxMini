## 快速开始

### 系统要求

- **macOS**：macOS 10.15 (Catalina) 或更高版本
  - 支持 Apple Silicon (M1/M2/M3) 和 Intel
- **Windows**：Windows 10 或更高版本（x64）
- **Linux**：Ubuntu 20.04、Fedora 35 或更高版本（x64）

### 硬件要求

- 本地语音识别：建议 8GB 以上内存
- 使用大模型（medium/large）：建议 16GB 以上内存
- 麦克风设备

### 下载安装

如果您不想编译，可以直接下载预编译版本：

1. 访问 [GitHub Releases](https://github.com/GravityPoet/ChordVox/releases)
2. 根据您的系统下载对应文件：
   - **macOS (Apple Silicon)**：`ChordVox-*-arm64.dmg`
   - **macOS (Intel)**：`ChordVox-*-x64.dmg`
   - **Windows**：`ChordVox-Setup-*.exe`
   - **Linux**：`ChordVox-*-linux-x86_64.AppImage` 或 `.deb` 或 `.rpm`

### macOS 首次启动

非 App Store 下载可能被 Gatekeeper 拦截，执行以下命令解除：

```bash
xattr -dr com.apple.quarantine /Applications/ChordVox.app
open /Applications/ChordVox.app
```

## 使用指南

### 首次设置向导

1. 启动应用后会自动进入 8 步设置向导
2. **步骤 1**：欢迎界面
3. **步骤 2**：麦克风权限设置
4. **步骤 3**：选择语音识别引擎
5. **步骤 4**：下载语音模型
6. **步骤 5**：设置语言（支持 58 种语言）
7. **步骤 6**：命名您的 AI 助手（如 "ChordVox"）
8. **步骤 7**：配置 AI 润色（可选，需要 API key）
9. **步骤 8**：设置热键（默认：macOS 用 Globe/Fn，Windows/Linux 用 \`）

### 基本使用方法

#### 1. 开始录音

**方式一：点按模式（Tap to Talk）**
- 按下热键一次开始录音
- 再次按下结束录音

**方式二：按住模式（Push to Talk）**
- 按住热键开始录音
- 松开热键结束录音
- *注：macOS 原生支持 Globe/Fn 键按住模式，Windows 需要原生键盘钩子*

#### 2. 语音识别流程

```
按下热键 → 开始录音 → 松开/再次按热键 → 本地识别 → [可选 AI 润色] → 粘贴到光标位置
```

#### 3. AI 助手指令模式

对您的 AI 助手说话，可以触发智能指令：

```
"Hi ChordVox，帮我起草一封商务邮件..."
"Hey Assistant，帮我总结一下会议内容..."
```

AI 会自动处理并生成相应文本，自动去除助手名字引用。

### 核心功能详解

#### 本地语音识别

ChordVox 提供三种本地语音引擎：

1. **whisper.cpp**（推荐）
   - 多种模型可选：tiny、base、small、medium、large、turbo
   - 平衡速度和质量推荐：base 或 small
   - 高质量推荐：medium 或 large

2. **NVIDIA Parakeet**
   - 基于 sherpa-onnx 的高效推理
   - 支持 25 种语言
   - 模型：parakeet-tdt-0.6b-v3（~680MB）

3. **SenseVoice**
   - 快速的语音识别引擎
   - 适合低配置设备

#### AI 润色配置

支持多个 AI 提供商：

| 提供商 | 模型 | 特点 |
|--------|------|------|
| **OpenAI** | GPT-5.2、GPT-5 Mini、GPT-4.1 | 最强推理能力 |
| **Anthropic** | Claude Sonnet 4.5、Haiku 4.5、Opus 4.5 | 平衡性能 |
| **Google** | Gemini 2.5 Pro、Flash 系列 | 高性价比 |
| **Groq** | LPU 加速模型 | 极快响应 |
| **本地** | GGUF 模型（Qwen、Llama、Mistral） | 完全离线 |

配置步骤：
1. 打开设置 → AI 润色
2. 选择 AI 提供商
3. 输入 API Key（会保存到 .env 文件）
4. 选择模型
5. 设置润色提示词（可选）

#### 双配置热键

设置两套独立的工作流：

- **配置 1**（快速草稿）：本地语音识别 + 快速模型（如 GPT-5 Nano）
- **配置 2**（精修输出）：高精度语音模型 + 高级 AI（如 GPT-5.2 或 Claude Opus）

分别设置不同的热键，一键切换工作模式。

#### 自定义词典

提升专业术语识别准确率：

1. 设置 → 自定义词典
2. 添加专业词汇：
   - 人名："张小明"
   - 技术术语："Transformer"、"CUDA"
   - 专业词汇："二尖瓣置换术"
   - 品牌名："ChordVox"

这些词汇会作为提示词传给语音识别引擎，大幅提升准确率。

#### 界面语言

支持 10 种界面语言：
- 中文简体、中文繁体
- English（英文）
- 日本語（日语）
- Deutsch（德语）
- Français（法语）
- Español（西班牙语）
- Português（葡萄牙语）
- Italiano（意大利语）
- Русский（俄语）

### 键盘快捷键

#### 默认热键

| 平台 | 热键 | 模式 |
|------|------|------|
| macOS | Globe 或 Fn | Push to Talk（按住） |
| Windows | \`（反引号） | Tap to Talk（点按） |
| Linux | \`（反引号） | Tap to Talk（点按） |

#### 自定义热键

1. 设置 → 热键设置
2. 选择 "更改热键"
3. 按下新的热键组合
4. 保存

支持的修饰键：
- `Command` / `Ctrl`
- `Option` / `Alt`
- `Shift`
- `Control`（macOS）

示例组合：
- `CommandOrControl+Shift+Space`
- `Alt+R`
- `Ctrl+Shift+F11`

### 系统权限要求

#### macOS

- **麦克风权限**：必须（录音用）
- **辅助功能权限**：必须（自动粘贴用）
- **可选权限**：录音权限（系统偏好设置）

权限设置步骤：
1. 打开系统设置 → 隐私与安全性
2. 麦克风：启用 ChordVox
3. 辅助功能：启用 ChordVox

#### Windows

- **麦克风权限**：必须（设置 → 隐私 → 麦克风）
- 无需辅助功能权限

#### Linux

- **X11**：需要 `xdotool`（自动粘贴）
- **GNOME Wayland**：自动配置系统快捷键
- **其他 Wayland**：需要 `wtype` 或 `ydotool`

安装粘贴工具：

```bash
# Debian/Ubuntu
sudo apt install xdotool

# Fedora/RHEL
sudo dnf install xdotool

# Arch Linux
sudo pacman -S xdotool

# Wayland (Sway/Hyprland)
sudo pacman -S wtype

# 通用 Wayland（需要 ydotoold 守护进程）
sudo apt install ydotool  # Debian/Ubuntu
sudo dnf install ydotool  # Fedora
```

### 存储空间管理

模型缓存位置：

- **macOS**：`~/.cache/openwhispr/`
- **Windows**：`%USERPROFILE%\.cache\openwhispr\`
- **Linux**：`~/.cache/openwhispr/`

子目录：
- `whisper-models/`：Whisper 模型
- `parakeet-models/`：NVIDIA Parakeet 模型
- `llama-models/`：本地 LLM 模型

清理方式：
1. 设置 → 存储空间
2. 选择要删除的模型
3. 点击 "卸载模型"

## 编译指南

### 环境准备

#### 1. 安装 Node.js

需要 Node.js 18 或更高版本（推荐 Node.js 20 LTS）

**macOS**（推荐使用 Homebrew）：
```bash
brew install node@20
```

**Windows**：
- 从 [nodejs.org](https://nodejs.org/) 下载安装

**Linux**（Debian/Ubuntu）：
```bash
curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
sudo apt install -y nodejs
```

验证安装：
```bash
node --version  # 应该 >= 18.x
npm --version   # 应该 >= 9.x
```

#### 2. 安装 Git

```bash
# macOS
brew install git

# Windows
# 从 https://git-scm.com/ 下载

# Linux
sudo apt install git  # Debian/Ubuntu
sudo dnf install git  # Fedora
```

#### 3. 克隆仓库

```bash
git clone https://github.com/GravityPoet/ChordVox.git
cd ChordVox
```

#### 4. 安装依赖

```bash
npm ci
```

注意：
- 使用 `npm ci` 而不是 `npm install`，确保安装锁定版本的依赖
- 如果失败，尝试：`rm -rf node_modules package-lock.json && npm ci`

### 编译步骤

#### 选项 1：快速开始（开发模式）

```bash
# 1. 编译原生模块
npm run compile:native

# 2. 下载语音引擎（当前平台）
npm run download:whisper-cpp
npm run download:sherpa-onnx
npm run download:llama-server

# 3. Windows 额外需要（其他平台可跳过）
npm run download:nircmd
npm run download:windows-key-listener
npm run download:windows-fast-paste

# 4. 启动开发模式
npm run dev
```

开发模式说明：
- 自动打开 Vite 开发服务器
- 启动 Electron 应用
- 支持热模块替换（HMR）
- 更改代码自动刷新

#### 选项 2：生产构建（当前平台）

```bash
# 1. 编译原生模块
npm run compile:native

# 2. 下载语音引擎
npm run download:whisper-cpp
npm run download:sherpa-onnx
npm run download:llama-server

# 3. Windows 额外需要
npm run download:nircmd
npm run download:windows-key-listener
npm run download:windows-fast-paste

# 4. 构建（根据平台选择）
npm run build:mac      # macOS
npm run build:win      # Windows
npm run build:linux    # Linux
```

构建输出位置：
- **macOS**：`dist/ChordVox-*-arm64.dmg` 或 `dist/ChordVox-*-x64.dmg`
- **Windows**：`dist/ChordVox-Setup-*.exe`
- **Linux**：`dist/*.AppImage`、`dist/*.deb`、`dist/*.rpm`

#### 选项 3：多平台构建

```bash
# 下载所有平台的语音引擎
npm run download:whisper-cpp:all
npm run download:sherpa-onnx:all
npm run download:llama-server:all

# 构建所有平台
npm run build
```

注意：
- 需要在每个平台上分别构建，不能交叉编译
- macOS 可以同时构建 arm64 和 x64（通用构建）
- Windows 需要在 Windows 上构建
- Linux 需要在 Linux 上构建

### 详细的编译脚本说明

#### 原生模块编译

项目包含几个需要编译的原生模块：

```bash
# 编译所有原生模块
npm run compile:native

# 分别编译（可选）
npm run compile:globe           # macOS Globe 键监听器（Swift）
npm run compile:fast-paste      # macOS 快速粘贴（Swift）
npm run compile:winkeys         # Windows 键监听器（C）
npm run compile:winpaste        # Windows 快速粘贴
npm run compile:linux-paste     # Linux 快速粘贴
```

注意：
- macOS 需要安装 Xcode Command Line Tools
- Windows 需要 Visual Studio Build Tools（或 MinGW-w64）
- 通常不需要手动编译，预编译二进制会自动下载

#### 下载外部资源脚本

```bash
# whisper.cpp 语音识别引擎
npm run download:whisper-cpp        # 当前平台
npm run download:whisper-cpp:all    # 所有平台

# sherpa-onnx（用于 NVIDIA Parakeet）
npm run download:sherpa-onnx        # 当前平台
npm run download:sherpa-onnx:all    # 所有平台

# llama.cpp 本地 LLM
npm run download:llama-server       # 当前平台
npm run download:llama-server:all   # 所有平台

# Windows 专用工具
npm run download:nircmd             # nircmd.exe（系统工具）
npm run download:windows-key-listener
npm run download:windows-fast-paste
```

### 环境变量配置

可选配置文件 `.env`：

```bash
# 复制示例文件
cp .env.example .env
```

编辑 `.env`：

```
# 调试日志级别（debug/info/warn/error）
OPENWHISPR_LOG_LEVEL=info

# 本地语音识别提供商（whisper/nvidia）
LOCAL_TRANSCRIPTION_PROVIDER=whisper

# Parakeet 模型

# API Keys（也可以在应用内设置）
OPENAI_API_KEY=sk-...
ANTHROPIC_API_KEY=sk-ant-...
GEMINI_API_KEY=...
GROQ_API_KEY=gsk_...

# GitHub Token（用于避免 API 限制，可选）
GITHUB_TOKEN=ghp_...
```

### macOS 代码签名（可选）

如果要分发应用，需要 Apple Developer 证书：

```bash
# 设置签名证书（build 会自动使用）
export CSC_NAME="Developer ID Application: Your Name (TEAM_ID)"
export CSC_IDENTITY_AUTO_DISCOVERY=true

# Notarization（公证）
export APPLE_ID="your@email.com"
export APPLE_APP_SPECIFIC_PASSWORD="xxxx-xxxx-xxxx-xxxx"
export APPLE_TEAM_ID="TEAM_ID"

# 构建并签名
npm run build:mac
```

不签名也可以运行，只是首次启动需要 Gatekeeper 解除隔离。

### Windows 代码签名（可选）

```bash
# 设置证书
export CSC_LINK="/path/to/certificate.pfx"
export CSC_KEY_PASSWORD="certificate-password"

# 构建并签名
npm run build:win
```

### 添加新功能

#### 添加新的 IPC 处理程序

1. 在 `src/helpers/ipcHandlers.js` 中添加处理函数
2. 在 `preload.js` 中暴露给渲染进程
3. 在组件中通过 `window.api.yourHandler()` 调用

示例：

```javascript
// ipcHandlers.js
ipcMain.handle('your-function', async (event, arg) => {
  // 处理逻辑
  return result;
});

// preload.js
yourFunction: (arg) => ipcRenderer.invoke('your-function', arg)

// 组件中
const result = await window.api.yourFunction('data');
```

#### 添加新的设置项

1. 在 `src/hooks/useSettings.ts` 中添加新的状态
2. 在 `src/components/SettingsPage.tsx` 中添加 UI
3. （可选）在 `src/helpers/` 中添加处理逻辑

#### 添加新的 AI 模型

1. 在 `src/models/modelRegistryData.json` 中添加模型定义
2. 在 `src/components/SettingsPage.tsx` 中更新 UI（如果需要）
3. 模型会自动出现在选择列表中

## 常见问题

### 编译相关问题

#### Q: npm install 失败？

尝试清理并重新安装：
```bash
rm -rf node_modules package-lock.json
npm cache clean --force
npm ci
```

#### Q: whisper.cpp 下载失败？

可能是 GitHub API 限制：

```bash
# 设置 GitHub Token（可选）
export GITHUB_TOKEN=ghp_your_token

# 或者手动下载
# 访问 https://github.com/ggerganov/whisper.cpp/releases
# 下载对应平台的二进制，放到 resources/bin/ 目录
```

#### Q: 编译 native 模块失败？

通常不需要手动编译，因为预编译二进制会自动下载。如果确实需要编译：

**macOS**：
```bash
xcode-select --install  # 安装 Xcode 命令行工具
npm run compile:globe
```

**Windows**：
安装 Visual Studio Build Tools（带 C++ 桌面开发组件）或 MinGW-w64。

#### Q: TypeScript 类型错误？

确保依赖正确安装：
```bash
npm ci
npm run typecheck
```

### 运行时问题

#### Q: 启动后麦克风没有反应？

1. 检查系统麦克风权限
2. 检查应用内是否选择了正确的输入设备
3. 尝试选择不同的麦克风
4. 开启调试日志查看：`--log-level=debug`

#### Q: 转录结果是空的？

1. 检查麦克风音量（需要足够大）
2. 检查音频输入设备是否正确
3. 尝试重启应用
4. 清除模型缓存：`rm -rf ~/.cache/openwhispr/whisper-models`

#### Q: 自动粘贴不工作？

**macOS**：
1. 检查辅助功能权限（系统设置 → 隐私与安全性 → 辅助功能）
2. 重启应用后重新尝试

**Windows**：
1. 尝试手动粘贴（Ctrl+V）看是否复制成功
2. 开启调试日志查看具体错误

**Linux**：
1. 安装粘贴工具：`sudo apt install xdotool`
2. Wayland 下尝试：`sudo apt install wl-clipboard wtype`

#### Q: macOS 提示应用损坏？

解除 Gatekeeper 隔离：
```bash
xattr -dr com.apple.quarantine /Applications/ChordVox.app
open /Applications/ChordVox.app
```

#### Q: 热键不工作？

1. 检查是否与其他应用的热键冲突
2. 尝试设置不同的热键
3. **macOS**：尝试重启（有些热键需要重启）
4. **Windows**：检查是否有其他程序占用了热键
5. **GNOME Wayland**：热键会注册到系统快捷键，可以在系统设置中查看

### 性能问题

#### Q: 本地识别很慢？

1. 使用更小的模型（如 tiny 或 base）
2. 尝试 NVIDIA Parakeet 引擎（sherpa-onnx）
3. 确保设备有足够的内存（至少 8GB）
4. 关闭其他占用 CPU 的程序

#### Q: AI 润色很慢？

1. 切换到更快的模型（如 GPT-5 Nano、Claude Haiku）
2. 尝试 Groq（LPU 加速）
3. 使用本地 LLM（如果网络慢）

### 获取帮助

如果问题仍未解决：

1. 开启调试模式并复现问题
2. 收集日志文件
3. 访问 [GitHub Issues](https://github.com/GravityPoet/ChordVox/issues) 提交问题
4. 加入 QQ 交流群（见 README）
